## IntroduÃ§Ã£o

O Strategy permite definir uma famÃ­lia de algoritmos, encapsulÃ¡-los e tornÃ¡-los intercambiÃ¡veis. Esse padrÃ£o permite que o algoritmo varie independentemente dos clientes que o utilizam. ## Elementos principais do Strategy [2]

- **Context (Contexto):** MantÃ©m uma referÃªncia para um objeto Strategy e interage com ele exclusivamente atravÃ©s da interface Strategy.

- **Strategy (EstratÃ©gia):** Interface comum para todas as implementaÃ§Ãµes de algoritmo suportadas.

### Contexto

Na aplicaÃ§Ã£o Eterna FGA, o padrÃ£o Strategy foi empregado para permitir diferentes estratÃ©gias de renderizaÃ§Ã£o de mÃ­dias digitais. O sistema exige que os usuÃ¡rios possam visualizar as mÃ­dias apresentadas de maneiras distintas â€” por exemplo, com qualidade superior ou adequada.

### MotivaÃ§Ã£o da escolha

O Strategy foi escolhido por permitir que diferentes comportamentos (neste caso, algoritmos de ordenaÃ§Ã£o) sejam definidos de maneira modular, limpa e intercambiÃ¡vel. A adoÃ§Ã£o desse padrÃ£o evita a utilizaÃ§Ã£o de estruturas condicionais complexas (como if ou switch) para selecionar o tipo de ordenaÃ§Ã£o, promovendo uma arquitetura mais flexÃ­vel, expansÃ­vel e de fÃ¡cil manutenÃ§Ã£o.
